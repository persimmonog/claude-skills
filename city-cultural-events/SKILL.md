# 同城文化活动推荐 Skill

## 工作流程

### 1. 确认用户需求

从用户输入中提取以下信息，缺少则主动询问：

| 信息 | 说明 | 示例 |
|------|------|------|
| 城市 | 必填 | 北京、上海、杭州 |
| 活动类型 | 可选，默认全部 | 展览、演唱会、话剧、脱口秀、音乐会 |
| 时间范围 | 必填，未提供则追问 | 本周末、五一、下个月、4月30日到5月5日 |
| 数量 | 可选，默认 20 | 推荐 10 个、最多 30 个 |

**时间范围处理规则**：
- 用户明确提及时间限定词时直接解析（如"本周末"→本周六日、"五一"→5月1日-5日、"下周"→下周一至周日）
- 用户未提及任何时间范围时，**必须追问用户**，例如："请问你想看哪个时间段的活动？比如本周末、五一假期、或具体日期？"
- 常用时间词解析参考：

| 用户说法 | 实际日期范围 |
|----------|-------------|
| 今天 / 今晚 | 当天 |
| 明天 | 明天 |
| 本周末 / 周末 | 本周六、周日 |
| 这周 / 本周 | 本周一至周日 |
| 下周 / 下周末 | 下周一至周日 / 下周六日 |
| 五一 / 五一假期 | 5月1日-5月5日 |
| 国庆 / 国庆假期 | 10月1日-10月7日 |
| 本月 / 这个月 | 本月1日至月末 |
| 下个月 | 下月1日至月末 |
| 最近 / 近期 | 今天起往后7天 |

### 2. 抓取活动数据

使用 bundled 脚本从大麦网、豆瓣同城和活动行获取实时活动数据：

```bash
python3 <skill-dir>/scripts/scrape_events.py <城市> [活动类型] [数量] [--format json|markdown]
```

**示例**：
- `python3 scripts/scrape_events.py 北京` — 获取北京全部活动，最多 20 条（JSON 输出）
- `python3 scripts/scrape_events.py 上海 展览 15 --format markdown` — 获取上海展览类活动，Markdown 表格输出

**活动类型参数**：全部（默认）、演唱会、话剧、音乐剧、音乐会、Livehouse、展览、脱口秀

> **注意**：脚本依赖 Playwright（`pip3 install playwright && playwright install chromium`）。如果环境没有安装，先安装再运行。

### 3. 解析脚本输出

脚本返回 JSON 格式，结构如下：

```json
{
  "city": "北京",
  "type": "全部",
  "search_date": "2026-04-29",
  "total": 20,
  "categories": {
    "展览": [...],
    "话剧/音乐剧": [...],
    "脱口秀/喜剧": [...],
    "演唱会/音乐节": [...],
    "其他": [...]
  }
}
```

每个活动包含：`title`（名称）、`date`（时间）、`venue`（场馆）、`price`（票价）、`tags`（标签/评分）、`ranking`（排行榜信息）、`link`（购票链接）

### 4. 按时间范围过滤

根据 Step 1 中确认的时间范围，对每个分类下的活动逐一过滤，只保留活动日期与用户指定时间范围有交集的活动。

**过滤规则**：
- 活动日期为单天（如 `2026.05.01`）：该天落在时间范围内则保留
- 活动日期为区间（如 `2026.04.30-05.05`）：与时间范围有任何一天重叠即保留
- 日期格式可能包含具体时间（如 `2026.05.01 周五 18:30`），提取年月日部分判断
- 豆瓣同城格式（如 `05月01日 周五 15:00-19:15`），同样提取日期判断
- 无法解析日期的活动保守保留，不丢弃

**无结果处理**：
- 如果过滤后某个分类没有活动，显示"该分类在指定时间范围内暂无活动"
- 如果所有分类都没有活动，告知用户"指定时间范围内暂无活动，是否扩大时间范围？"

**输出说明**：
- 在推荐结果开头明确标注时间范围，如："📅 时间范围：2026年5月1日-5月5日（五一假期）"
- 被过滤掉的活动不展示在任何输出中（推荐列表和地图均不出现）

### 5. 格式化输出

#### 5.1 手机端排版原则

输出文本主要在手机 app 内阅读，需遵循以下原则：

- **紧凑优先**：每个活动控制在 3 行以内，减少滚动
- **信息合并**：时间 + 地点放同一行，票价 + 来源放同一行
- **避免深缩进**：不使用多级嵌套列表，所有信息扁平排列
- **行宽适中**：每行尽量控制在 40 字以内，避免手机端换行错乱
- **空链接省略**：链接为空时直接不展示该行，不写"暂无"

#### 5.2 输出模板

整体结构按以下顺序展示：

```
# {城市}文化活动推荐

> 📅 {时间范围标注}
> 数据来源：大麦网、豆瓣同城、活动行

---

## {分类emoji} {分类名}

**1. {活动名称}** 🏆{排行榜/评分标注}
📍 {date} · {venue}
🎫 {price}

**2. {活动名称}**
📍 {date} · {venue}
🎫 {price}

> 💰 票价区间：¥{最低价} ~ ¥{最高价}

---

## 🔥 热门活动推荐

1. **{活动名称}** — {排行榜信息}
2. **{活动名称}** — {排行榜信息}

## 🎫 购票提示

- {提示内容}
```

#### 5.3 格式细节

- 活动序号从 1 开始编号，加粗 + 序号 + 名称组合成一行标题
- 排行榜/评分标签紧跟名称之后（🏆 标注）
- **信息行合并**：日期和地点用 `·` 分隔放同一行（📍 前缀），票价单独一行（🎫 前缀）
- 如有购票链接且不为空，在票价行末尾加 ` · [购票]({link})`
- 每个分类内的活动用 `---` 分隔
- 分类按顺序排列：展览 → 话剧/音乐剧 → 脱口秀/喜剧 → 演唱会/音乐节 → 其他
- 分类之间用 `---` 水平线分隔
- 分类 emoji 映射：话剧/音乐剧 → 🎭、脱口秀/喜剧 → 🎤、演唱会/音乐节 → 🎵、展览 → 🖼️、其他 → 🎪

#### 5.4 票价区间

每个分类末尾添加一行（引用格式）：
```
> 💰 票价区间：¥{最低价} ~ ¥{最高价}
```

#### 5.5 热门活动 & 购票提示

热门活动推荐放在所有分类列表之后：
```
## 🔥 热门活动推荐

1. **{活动名称}** — {排行榜信息}
2. **{活动名称}** — {排行榜信息}
```

购票提示紧随其后：
```
## 🎫 购票提示

- {提示内容}
```

### 6. 附加建议

在推荐列表后添加购票提示时，注意：
- 如有"可选座"、"限时早鸟"、"条件退"等标签则在购票提示中提醒
- 价格极低的活动（如¥8起、¥39起）可作为"性价比之选"单独提及

### 7. 生成活动地图

在推荐结果输出后，自动生成一张哈利波特活点地图（Marauder's Map）风格的交互式 HTML 地图。

#### 7.1 准备地图数据

从 Step 3 的 JSON 数据中提取活动，转换为地图所需的格式：

```json
[
  {"name": "活动名称", "venue": "场馆地址", "date": "日期", "type": "concert", "lat": 23.13, "lon": 113.32},
  ...
]
```

字段映射规则（按输出优先级排列，"其他"放最后）：

| scrape_events 分类 | type 值 |
|---|---|
| 脱口秀/喜剧 | comedy |
| 话剧/音乐剧 | theater |
| 展览 | exhibition |
| 演唱会/音乐节 | concert |
| 其他 | other |

> 注：地图标记统一为红色图钉样式，不再按分类区分颜色。

#### 7.2 Geocoding

利用 Claude Code 自身的地理知识，为每个场馆地址添加 `lat`/`lon` 经纬度字段。
无需调用外部 API，直接在对话中完成坐标匹配。

注意：
- 中国大陆知名场馆坐标精度较高（偏差几十到几百米）
- 小众私人场地坐标可能只到大致街区级别
- 地址无法识别的活动跳过，不在地图中标注

#### 7.3 生成 HTML

执行 `scripts/generate_html.py`：

```bash
python3 <skill-dir>/scripts/generate_html.py --events '<JSON>' --output /mnt/user-data/outputs/marauders_map.html
```

生成的 HTML 是基于 Leaflet + OpenStreetMap 的单页应用，包含：
- 地图瓦片羊皮纸滤镜（`.leaflet-tile-pane` 上施加 sepia/contrast/brightness 叠加，不影响标记）
- 暗色晕影（radial-gradient vignette）
- 红色图钉标记（📌 样式，SVG 绘制，红色渐变头部 + 金属针杆，无字母标注）
- 点击弹出活动详情卡片
- 左下角类型图例（同款式红色图钉）

> **滤镜重要**：羊皮纸滤镜必须只加在 `.leaflet-tile-pane` 上，不能加在 `#map` 容器上，否则会把图钉也滤成棕色。

#### 7.4 展示结果

将生成的 HTML 文件展示给用户，附说明："在浏览器中打开即可查看，点击标记可看活动详情，支持缩放。"

## 注意事项

- **数据时效性**：在推荐结果开头注明搜索日期，提醒用户活动信息可能变化
- **无结果处理**：如果某个分类没有找到活动（或时间过滤后为空），明确说明"该分类暂无活动"
- **信息准确性**：所有活动信息来自大麦网实时数据，不编造任何内容
- **链接处理**：购票链接为空时直接省略该行，不展示"暂无"（手机端减少冗余信息）
